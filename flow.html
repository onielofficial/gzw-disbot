0 · Prerequisites

Before extraction. Confirm the kit before you move out.

CHECK Python 3.11 or newer installed
discord.py 2.4 needs Python 3.10+. We test on 3.11.
python --version
# python 3.11.x or higher
CHECK Outbound HTTPS to discord.com + gzwtacmap.com
If you're behind a firewall, allowlist these origins.
CHECK A Discord guild where you are an admin
For fast slash-command sync put its ID in DEV_GUILD_IDS.
Note · The bot does not need any Discord privileged intents. Slash commands only.

1 · Install

Get the code on disk. ~30 seconds with cached pip.

STEP Clone the repo & create a venv
git clone <your-fork-or-this-repo> gzw-disbot
cd gzw-disbot
python -m venv .venv
. .venv/bin/activate         # Windows: .venv\Scripts\activate
STEP Install dependencies
pip install -r requirements.txt
Tip · On Alpine / minimal Linux you may need apt-get install gcc libxml2 for lxml first.

2 · Discord application

Stand up a Discord bot identity. Re-runnable any time.

STEP Create a new Application
Open https://discord.com/developers/applications → New Application.
STEP Bot tab → Reset Token
Copy the token. Treat it like a password — it goes in .env only.
Sensitive · Don't commit .env. .gitignore excludes it by default.
STEP Build an OAuth invite URL
OAuth2 → URL Generator → scopes: bot, applications.commands. Bot Permissions: Send Messages, Embed Links, Use Slash Commands. Invite to your test guild.

3 · Configure .env

One file controls every knob.

STEP Copy the template
cp .env.example .env
STEP Fill in DISCORD_TOKEN and DEV_GUILD_IDS
DISCORD_TOKEN=MTI3...
DEV_GUILD_IDS=123456789012345678
GZW_BASE_URL=https://gzwtacmap.com
REFRESH_CRON=0 */6 * * *
SCRAPE_CONCURRENCY=4
SCRAPE_DELAY_MS=400
Tip · Leave DEV_GUILD_IDS empty later for global sync (~1h to propagate).

4 · Seed snapshot (offline-friendly)

Optional but recommended for first boot. Lets the bot answer immediately.

STEP Run the seeder against the bundled fixtures
python scripts/seed_from_fixtures.py
Writes data/snapshot.json with sample tasks/keys/places — enough to demo every command without scraping.
STEP (Alt) Scrape gzwtacmap.com once before booting
python scripts/scrape.py
Polite · The scraper caps concurrency at 4 and waits ~400 ms between requests.

5 · Run the bot

Boots the asyncio loop, syncs commands, schedules refresh.

STEP Start in foreground
python -m bot.main
Look for synced N commands and refresh ok — {…} on the console.

6 · Verify in Discord

Eyes-on inside your guild.

CHECK /help renders the briefing embed
Olive-themed embed, banner header, command grid in a codeblock.
CHECK /task search query:incognito returns the task
CHECK /place search query:1373 resolves by ID
CHECK /key search autocomplete suggests keys
CHECK /refresh works (owner only)

7 · Promote to production

Pick one. systemd, Docker, or a managed runtime.

OK systemd unit (Linux)
sudo cp deploy/gzw-disbot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now gzw-disbot
journalctl -u gzw-disbot -f
See docs/DEPLOYMENT.md for the unit file template.
OK Docker
docker build -t gzw-disbot .
docker run --rm --env-file .env -v $PWD/data:/app/data gzw-disbot
CHECK Set DEV_GUILD_IDS= empty for global sync
After verifying in your dev guild, switch to global. Allow ~1h to propagate.

8 · Operator tracker (optional but fun)

Per-user task & key progress, persisted in SQLite.

USE Mark a task as done
/track set kind:task target:incognito status:done
USE Add a key to your wallet
/track set kind:key target:tiger-bay-armory status:owned
USE Show your status panel
/track me
Privacy · Tracker replies are ephemeral — only you see them.

9 · Troubleshooting

If you see this, do that.

FAIL "Missing required env var: DISCORD_TOKEN"
Edit .env, set DISCORD_TOKEN=…, re-run.
FAIL "synced 0 commands"
Either DEV_GUILD_IDS doesn't include your guild, or you're using global sync (give it 1h).
FAIL "fetch giving up: 403"
gzwtacmap.com is rate-limiting or blocking the User-Agent. Lower SCRAPE_CONCURRENCY and bump SCRAPE_DELAY_MS; set a contact email in USER_AGENT.
FAIL "snapshot empty — running initial scrape" loops
Scraper found nothing. Run python scripts/seed_from_fixtures.py as a fallback while you debug parsing.
FAIL Discord embed says "Comms broken"
Means a cog returned a known error embed. Check the bot console — full traceback is logged.

11 · Bot data flow

How a slash command turns into an embed.

   user types /task search query:incognito
                │
                ▼
   ┌─────────────────────┐
   │ Tasks cog (cogs/    │
   │ tasks.py)           │
   └──────────┬──────────┘
              │ store.task_by_slug or fuzzy search
              ▼
   ┌─────────────────────┐
   │ GZWStore (in-mem)   │  ←── snapshot.json (loaded at boot)
   └──────────┬──────────┘
              │ dict
              ▼
   ┌─────────────────────┐
   │ embeds.embed_task() │  → discord.Embed
   └──────────┬──────────┘
              │
              ▼
        Discord channel

  Background:
    apscheduler ──▶ scrape_all() ──▶ writes new snapshot
                                  ──▶ atomically swaps GZWStore

12 · Command quick reference

Cheat-sheet you can leave open during ops.

/help                                       → briefing
/task search query:<name>                    → one task
/task list trader: type: faction: limit:    → filtered list
/key search query:<name>                     → one key + doors
/key list region: limit:                    → keys per region
/place search query:<name|id>                → POI by id (1373) or name
/place list region: group: limit:           → POIs filtered
/lz                                          → all landing zones
/cop                                         → all combat outposts
/trader info name:<n>                        → vendor dossier
/faction info name:<n>                       → faction dossier
/faction list type:<pmc|hostile>             → faction list
/search query:<text> limit:                  → universal fuzzy search
/track me                                    → your operator panel
/track set kind:<task|key> target:… status:… → mark progress
/track faction name:<LRI|Mithras|CSI>        → set your PMC
/track reset                                 → wipe your tracker
/map                                         → map link + cache stats
/refresh                                     → owner-only re-scrape
/ping                                        → latency check
